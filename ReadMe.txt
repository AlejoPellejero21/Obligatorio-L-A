MANUAL DEL PROYECTO [ALEJO PELLEJERO && LUCAS ALVAREZ]

1. Nombre de ARCHIVOS JS || HTML|| CSS 

Para los nombre de archivos utilizamos como regla general del proyecto colocar el nombre en Upper Camel Case y al final del mismo (.A&L.js) .

Por ejemplo: Example.Mini.A&L.js || ExampleExhtml.A&L.js


2. Nombre de VARIABLES

Para los nombres de las variables utilizaremos el famoso Camel case.

Por ejemplo: soyUnaVariableUtil || soyOtraVariable


3. Nomres de CONSTANTES

Para los nombres de las constantes utilizaremos Upper Camel Case.

Por ejemplo: SoyUnaConstante || EstoEsUnaConstante


4. Nombre de BRANCHES / RAMAS

Para los nombres de las ramas utilizaremos dos tipos 'feature' para cuando es algo nuevo agregar e 
'issue' para cuando se trabaja sobre algo ya subido.

Seguido del tipo de rama se agregara una barra '/' y luego el nombre de la funcionalidad a trabajar.

Por ejemplo: feature/nombreDeLaFuncionalidad || issue/nombreDeLaFuncionalidad

En el caso de que la rama ya exista y se quiera crear otra para la misma funcionalidad se le agregara versionado al final.

Por ejemplo: feature/nombreDeLaFuncionalidadV2 || feature/nombreDeLaFuncionalidadV3


5. Main index.html

En el index.html estara colocado todo el html respectivo a cada funcionalidad y ademas dentro de las extensiones
estaran cada una de las funcionalidades por separado.


6. Nombres de CARPETAS

Los nombres de carpetas se escribiran en Upper Cammel Case.

Por ejemplo: ExampleDeCarpeta || OtroExampleCarpeta
